Conceitos de Git e GitHub

Um guia prático para iniciantes

O que é Git?

Git é um sistema de controle de versão distribuído que registra mudanças em arquivos ao longo do tempo. Ele permite criar commits, trabalhar em branches e restaurar versões anteriores.

  • Distribuído: cada máquina tem um repositório completo.
  • Rápido e seguro: usa hashes para identificar commits.
  • Flexível: suporta diferentes fluxos de trabalho.

O que é GitHub?

GitHub é uma plataforma de hospedagem de repositórios Git que adiciona recursos de colaboração como Issues, Pull Requests, Actions (CI/CD) e revisão de código.

Importante: Git é a ferramenta de versionamento; GitHub é o serviço que usa Git para colaboração.

Termos Fundamentais

Repositório
Local onde o projeto e seu histórico de commits ficam armazenados.
Commit
Um snapshot do projeto com mensagem, autor e data.
Branch
Linha de desenvolvimento independente dentro do repositório.
Merge
Combina mudanças de uma branch em outra.
Remote
Repositório remoto, normalmente hospedado no GitHub.
Clone
Cópia de um repositório remoto para sua máquina.
Fork
Cópia de um repositório no GitHub para sua conta.
Pull Request
Proposta de integração de alterações em um repositório.
.gitignore
Arquivo que lista os itens que o Git deve ignorar.

Comandos Essenciais

Inicializar e configurar

git init
git config user.name "Seu Nome"
git config user.email "voce@email.com"

Salvar mudanças

git status
git add arquivo.txt
git commit -m "Mensagem clara"

Trabalhar com repositórios remotos

git remote add origin https://github.com/usuario/repositorio.git
git push -u origin main
git pull origin main

Branches

git branch feature/conceitos
git checkout feature/conceitos
git checkout -b feature/conceitos

Fluxo Básico de Trabalho com Git e GitHub

  1. 1. Criar um repositório local com git init.
  2. 2. Fazer alterações nos arquivos do projeto.
  3. 3. Salvar as mudanças com git add e git commit.
  4. 4. Conectar ao GitHub com git remote add origin.
  5. 5. Enviar para o repositório remoto com git push.
  6. 6. Colaboradores podem baixar as atualizações com git pull.
  7. 7. Para novas funcionalidades, criar uma branch, trabalhar nela e depois integrar via merge ou pull request.
